Drupal a ZF

15 február, 2011 - 14:58 poslal blueminds

Drupal je pre mna mimoriadne zaujimavy z jedneho dovodu. Poskytuje taku mieru flexibility ako ziadny iny web „framework“ (pre 7 uz bez uvodzoviek), s ktorym som mal tu cest. Co vsak ked vas pritlaci a potrebujete nejake ozajstne nastroje?

ZF (http://framework.zend.com/) ma okolo seba viac komercneho diania ako je zdrave, fakt vsak je, ze jeho kvality su chvilami nedocenitelne. Ked este pozriete na tu fajnovu licenciu, tak krizencovi geeka a projektmanagera zacnu tiest nie len sliny. Znamena to teda, ze zabudnime na Drupal a hor sa na ZF? No nie… Ved Drupal okrem spominanej flexibility poskytuje aj nie malo hotovej funkcionality. Nuz tu je recept na to, ako podvadzat Drupal, bez toho aby sme mu krivdili:

  1. V adresari /sites/all vytvorte adresar libraries, ak tak uz nebolo vykonane a nakopirujte tam celeho zenda, samozrejme do adresara ‚Zend‘.
  2. V module, ktory ma vyuzivat ZF fiiczury, respektive v nejakom bootstrapovskom/u­tilovskom, ktory pouzivate pre kazdy projekt, na zaciatok supnite tento kod:
if (file_exists('sites/all/libraries/Zend')) {
  $libraries = explode(PATH_SEPARATOR, get_include_path());
  $libraries[] = 'sites/all/libraries';

  set_include_path(implode(PATH_SEPARATOR, $libraries));

  require_once('Zend/Loader.php');
}

Tychto par riadkov prida kniznicu do „PHP Include Paht“, co je podmienkou pre pouzitie tej alebo onej kniznice zo ZF, a da k dispozicii Loader triedu, pomocou ktorej to viete spristupnit.

  1. Ako na vec. Ak chcete pouzit niektoru z kniznic, musite ju pridat (neviem normalnejsi vyraz v sk, snad mi rozumiete…). Spravite to nasledovne:
Zend_Loader::loadClass('Zend_Gdata_ClientLogin');  Zend_Loader::loadClass('Zend_Gdata_Calendar');

Kod hore nam pridal dve kniznice. Prva na autentifikaciu do sluzieb Google, druha je implementaciou Google Calendar API. No a co dalej?

  1. Vieme uz ako spristupnit fiiczury ZF. Je tu ale este par veci. Kedze Drupal nam poskytuje zakladne zalezitosti okolo zivotneho cyklu stranky (ja viem, zneje to hrozne, ide o ‚page lifecycle‘), nie je nam zapotreby pouzivat MVC zo ZF. Tiez mame v Drupale celkom schopne riesene pristupove prave, cize ACL a AMF nam tiez az tak netreba…, atd. Vypadne nam z toho to, ze zo ZF potrebujeme akurat veci typu svajcnozika. Napr. implementacie API roznych sluzieb, krypotvanie, pdf, pripadne ine srandicky. Inac povedane, potrebujeme vyuzivat ZF iba lokalne, a nie globalne. Mne sa preto osvedcilo nasledujuce:

4a) Ak potrebujem vyuzit nejake fiiczury ZF, spravim to v ramci jednej funkcie, respektive niekolkych… vid prikad:

function gcal_event_add($account, $event) {
  Zend_Loader::loadClass('Zend_Gdata_ClientLogin');
  Zend_Loader::loadClass('Zend_Gdata_Calendar');

  $client = Zend_Gdata_ClientLogin::getHttpClient($account['email'], $account['pass'], Zend_Gdata_Calendar::AUTH_SERVICE_NAME);
  $service = new Zend_Gdata_Calendar($client);

  $gcal_event = $service->newEventEntry();
  $gcal_event->title = $service->newTitle($event->title);

  $gcal_event->where = array($service->newWhere($event->where));

  $gcal_event->content = $service->newContent($event->description);

  $when = $service->newWhen();
  $when->startTime = $event->getStart();
  $when->endTime = $event->getEnd();
  $gcal_event->when = array($when);

  return $service->insertEvent($gcal_event);
}

function gcal_event_update($account, $event) {
  Zend_Loader::loadClass('Zend_Gdata_ClientLogin');
  Zend_Loader::loadClass('Zend_Gdata_Calendar');

  $client = Zend_Gdata_ClientLogin::getHttpClient($account['email'], $account['pass'], Zend_Gdata_Calendar::AUTH_SERVICE_NAME);
  $service = new Zend_Gdata_Calendar($client);

  $gcal_event = $service->getCalendarEventEntry($event->event_link);

  $gcal_event->title = $service->newTitle($event->title);
  $gcal_event->where = array($service->newWhere($event->where));
  $gcal_event->content = $service->newContent($event->description);

  $when = $service->newWhen();
  $when->startTime = $event->getStart();
  $when->endTime = $event->getEnd();
  $gcal_event->when = array($when);

  return $gcal_event->save();
}

function gcal_event_delete($account, $event) {
  Zend_Loader::loadClass('Zend_Gdata_ClientLogin');
  Zend_Loader::loadClass('Zend_Gdata_Calendar');

  $client = Zend_Gdata_ClientLogin::getHttpClient($account['email'], $account['pass'], Zend_Gdata_Calendar::AUTH_SERVICE_NAME);
  $service = new Zend_Gdata_Calendar($client);

  $gcal_event = $service->getCalendarEventEntry($event->event_link);
  $gcal_event->delete();
}

A aby neboli nejasnosti, $event je len helper trieda, ktorou som si zjednodusil zivot:

class GcalEvent {
  public $recurrence;
  public $start;
  public $end;
  public $properties;
  public $title;
  public $description;
  public $where;
  public $event_link;

  function getStart() {
    return date('Y-m-d\TH:i:sP', $this->start);
  }

  function getEnd() {
    return date('Y-m-d\TH:i:sP', $this->end);
  }
}

4b) Ak ste zatial D6, nezabudnite, ze ZF vyhadzuje vynimky – je ich treba osetrit v try – catch.
