Existuje nieco co dokaze analyzovat databazu, najst problem a opravit ho?

Uz par tyzdnov mam taky problem s databazou. V podstate sa to zacalo niekedy po aktualizacii Drupalu na 6.19 (Update.php pouzivam). Stranku mam na ic.cz. Podozrievam hosting, ze im vypadava server a nasledne sa po*e*e nieco v databaze. 2 mesiace to islo vpohode a nebol problem.

Napise mi to, ze nemam pristup k stranke. Nezalezi an aky link kliknem ci som prihlaseny alebo nie. No zaroven obcas ide dobre a vyzera v poriadku. Ked nastane chyba, tak sa stranka zobrazuje skoro cela. Zobrazuje hlavicku, menu (ale ma menej odkazov), paticku. Namiesto stlpcov a obsahu napise chybovu hlasku, kde oznaci zopar modulov a inych suborov Drupalu. Vacsinou su to drupalovske subory a teda nie doinstalovane moduly.
Ked si cez phpMyAdmin dam skontrolovat databazu, tak najde problem v "mojadatabaza.users". Ked som dal opravit problem, tak stranka isla, ale ked som sa prihlasil, tak sa to potentovalo. Druhy karat, ked som to dal opravit, to uz nepomohlo.

Som začiatočník, skúsil som to s tým riadkom do .htaccess, bolo to veľa pokusov a omylov, ale na koniec sa to podarilo. Pre podobných začiatočníkov ako ja sem dám, ako vyzerá časť môjho .htaccess po úprave:

# Override PHP settings. More in sites/default/settings.php
# but the following cannot be changed at runtime.

# PHP 4, Apache 1.

php_value magic_quotes_gpc 0
php_value register_globals 0
php_value session.auto_start 0
php_value mbstring.http_input pass
php_value mbstring.http_output pass
php_value mbstring.encoding_translation 0
php_value session.save_handler user

# PHP 4, Apache 2.

php_value magic_quotes_gpc 0
php_value register_globals 0
php_value session.auto_start 0
php_value mbstring.http_input pass
php_value mbstring.http_output pass
php_value mbstring.encoding_translation 0
php_value session.save_handler user

# PHP 5, Apache 1 and 2.

php_value magic_quotes_gpc 0
php_value register_globals 0
php_value session.auto_start 0
php_value mbstring.http_input pass
php_value mbstring.http_output pass
php_value mbstring.encoding_translation 0
php_value session.save_handler user

Keď som to urobil, smozrejme chyba sa zobrazovala stále. Skúsil som v PgSql Administracii vyprázdniť v tabuľke všetky tabuľky cache až cache_views_data (11 tabuliek) a stránka začala fungovať. Ešte blbne vstup do administrácie, ale už ide aspoň pridať fotky, članky , stránky a upravovať pôvodne vytvorené.

Pri vstupe do administrácie dáva nasledovné:

Webová lokalita nemôže zobraziť túto stránku.
HTTP 500
Najpravdepodobnejšie príčiny:
•Na webovej stránke prebieha údržba.
•Webová lokalita obsahuje chybu programovania.

Hádam toto pomôže takým amatérom ako ja a ušetrí im to cestu pokusov.
