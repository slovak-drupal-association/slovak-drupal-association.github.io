MOJA INSTALACIA DRUPALU A PROBLEMY S KTORYMI SOM SA STRETOL

30 november, 2010 - 17:34 poslal armix

-na serveri v directory- public.html som si vytvoril novy dir. a nazval som ho ho - a. Do tohto dir. Som potom uploadol drupal.V prehliadaci(Mozilla,IE atd.) som zadal www.moja-domena.sk/a/install.php a mala by sa mi spustit instalacia drupal.Nestalo sa tak,a dovod som sa dozvedel az po kontaktovani supportu mojho hosting providera.Moj provider ma na svojich serveroch zakazane OPTIONS.Vypol som ich tak,ze som si na serveri v priecniku public.html/a nasiel document .htaccess Musel som povolit jeho editaciu, aby som sa do neho dostal.V tomto dokumente som potom pred kazdy text,zacinajuci sa slovom options vlozil symbol # a nakoniec som tento document ulozil.Ked teraz restartujem prehliadac s mojou strankou,naskocila mi instalacia drupal.Netesil som sa vsak dlho
-dal som si install drupal in English
-dalsi problem,ktory sa mi naskytol bol,ze pri tomto kroku mi instalacia vyhodila chybu
The Drupal installer requires that you create a settings file as part of the installation process.
1. Copy the ./sites/default/default.settings.php file to./sites/default/settings.php.
2. Change file permissions so that it is writable by the web server. If you are unsure how to grant file permissions, please consult the on-line handbook.
More details about installing Drupal are available in INSTALL.txt.
Tuna som si na ploche v notepade vytvoril prazdny document.Otvoril som ho a dal som file/save as alebo document/ulozit ako.Tuna je velmi dolezite,aby som zmenili save as type Text document (*txt)na all files inak to nebude fungovat.Potom som document nazval settings.php
tento prazdny php document som nasledne nahral na server tam,kde mam drupal(co je v mojom pripade Publi_html/a. Tu som si nasiel zlozku sites a v nej default..Po tom,ako som ho uploadol ,zmenil som jeho permissions na 666.V mojom ftp paneli to bolo pod moznostou Chmod.Po restartovani prehliadaca bol,problem odstraneny.Zase som sa ale tesil menej ako 20 sekund.
Hned na dalsom kroku mi totizto vyskocil dalsi error.
register_globals is enabled. Drupal requires this configuration directive to be disabled. Your site may not be secure when register_globals is enabled. The PHP manual has instructions for how to change configuration settings. (Currently using PHP register globals Enabled ('1'))
Na tomto som sa natrapil asi najviac,ale nakoniec som prisiel na dve riesenia.Jedno,pre mna nefungovalo,ale s videi na youtube to vyzera,ze vela ludom pomohlo.Je celkom jednoduche.Vytvoril som v notepade novy document a napisal som do neho register_globals = Off a nazval som ho php.ini (zase je dolezite zmenit saved file types na all files,nech sa to neulozi ako txt document,ale ako php) potom som tento document uploadol na server do dir.kde som mal drupal,pre mna to bolo public.html/a
Druhy sposob,ktory som musel aplikovat ja je taky,ze hlboko v /conf/php52 je document php.ini a v nom som zmenil register globals s on na off.
-v dalsom kroku som si nakonfiguroval databazu,v ktorej bude drupal pracovat.Database name a database username je vacsinou to iste.Nazov mojej databazy,ktoru som si vytvorili v cPaneli v sprave detabaz MySQL.Heslo som si tam vytvoril tiez, takze som ho len napisal .Tu pozor.Musel som si otvorit ponuku Advanced options Drupal instalacia ma nadstaveneho databazoveho hosta na localhost.Toto som musel prepisat na databaze host mojej databazy.
-na dalsej strane som si drupal nakonfiguroval.Zadal som emailovu adresu,meno,heslo,odklikol som clean URL a odoslal som to.
