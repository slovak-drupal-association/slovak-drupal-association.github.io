Rules + Content Access = Chyba

24 máj, 2012 - 17:06 poslal Xicht

Dobrý deň, pomocou Rules kopírujem obsah políčka (desatinné číslo) do iného políčka (tiež desatinné číslo) – obidva políčka vo vytváranom node.

Mám zadefinované: Po uložení nového obsahu – set data value: field_a >> field_b.

Funguje to ako má až kým nezapnem modul „Content Access“ alebo „Node access node reference“ alebo aj „Node access user reference“

Po zapnutí hociktorého z uvedených modulov dostanem hlášku:

PDOException: SQLSTATE[23000]: Integrity constraint violation: 1062 Duplicate entry ‚28–0-all‘ for key ‚PRIMARY‘: INSERT INTO {node_access} (nid, realm, gid, grant_view, grant_update, grant_delete) VALUES (:db_insert_pla­ceholder0, :db_insert_pla­ceholder1, :db_insert_pla­ceholder2, :db_insert_pla­ceholder3, :db_insert_pla­ceholder4, :db_insert_pla­ceholder5); Array ([:db_insert_pla­ceholder0] ⇒ 28 [:db_insert_pla­ceholder1] ⇒ all [:db_insert_pla­ceholder2] ⇒ 0 [:db_insert_pla­ceholder3] ⇒ 1 [:db_insert_pla­ceholder4] ⇒ 0 [:db_insert_pla­ceholder5] ⇒ 0 ) v node_access_wri­te_grants() (riadok 3417 z /var/www/html­/m3/modules/no­de/node.module).

Keď v rules zmením udalosť na „Pred uložením obsahu“ tak to funguje, ale ja potrebujem zapisovať až „Po uložení nového obsahu“, lebo políčko z ktorého kopírujem bude „computed field“ a pred uložením ešte nie je vypočítané…

Typ obsahu v ktorom to kopírujem má povolené v oprávneniach vytvorenie, upravenie, odstránenie ľubovoľného obsahu pre všetkých používateľov vrátane anonymných a napriek tomu to nefunguje.

Má niekto nápad, kde hľadať chybu?

Skúšam to na čistej experimentálnej inštalácií Drupalu 7 s čerstvými verziami uvedených modulov.
