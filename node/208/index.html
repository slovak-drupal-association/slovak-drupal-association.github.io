Pass parameter cez URL

14 august, 2007 - 20:51 poslal sketman

Zdravim,

mam v PHP naprogramovanu stranku (nazvime ju mainpage), ktora dynamicky vylistuje vo formate tabulky data z databazy. Kazdy riadok z prveho stlpca tabulky je klikatelny, pricom shortcut ukazuje na druhu stranku (nazvime ju detailpage), kde su uvedene podrobnejsie informacie k prislusnemu riadku v tabulke mainpage. Identifikator prislusneho zaznamu ktoreho detaily si prajeme vidiet je z mainpage posielany cez URL (formou GET) do detailpage.

Aby som tuto funkcionalitu mohol zakomponovat do Drupalu, vytvoril som v Drupali dve stranky, typ page, input format PHP. Prva stranka (mainpage) mi funguje bez problemov, vylistuje mi tabulku s nacitanymi udajmi, riadky v prvom stlpci tabulky su klikatelne. Problem vsak je, ze ked kliknem na shortcut, Drupal vypise „Page not found“. Nevie najst stranku detailpage, vytvorenu v Drupali. Predpokladam, ze je to preto, lebo som asi pouzil nespravnu syntax v URL. Na mojom testovacom servri, mimo Drupalu, to totiz funguje.

Takze, takto vyzera dotycny shortcut:

<a href="?q=node/23?mena_ucastnikov_link=<?php echo $row_mena_ucastnikov['priezv_a_meno']; ?>"><?php echo $row_mena_ucastnikov['priezv_a_meno']; ?></a>

pricom: node/23 je stranka detailpage kam sa posiela ID zaznamu v tabulke a „mena_ucastni­kov_link“ je premenna v ktorej sa prenasa ID prislusneho zaznamu z mainpage do detailpage.

Velmi by mi pomohlo, keby mi niekto pomohol prist na to, preco to nefunguje, aka syntax by sa mala pouzit.

Inak tvoj problem s URL – v drupale su uz vsetky cesty castou URL tak dostat dalsi parameter do URL staci takto:

Ale ako pisem treba to robit cez funkciu l() ktora zabezpeci vzdy spravne vygenerovanie odkazu (napr pri peknom url).

V kazdom pripade, hodil som moju otazku aj na ofialne Drupal.org forum, kde mi niekto poradil nasledovny format URL:

Na zaklade toho som zmenil svoj kod nasledovne:

<td><a href="?q=node/23&mena_ucastnikov_link=<?php echo $row_mena_ucastnikov['priezv_a_meno']; ?>"><?php echo $row_mena_ucastnikov['priezv_a_meno']; ?></a></td>

Toto je uz lepsie, lebo nevypisuje page not found, ale objavi sa pozadovana stranka detailpage s tabulkou. Problem vsak je, ze tabulka nie je vyplnena, data sa nenacitaju. Cize zda sa, ze premenna mena_ucastnikov_link neprenasa pozadovane ID zaznamu tak ako ma…:(
