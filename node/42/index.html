upravenim a zjednodusenim snippetu z drupal.org som do casti s page.tpl.php pridal takyto kusok kodu ktory mi dal newsy presne ako som potreboval
<h2>News</h2>
<?php
$tid = 1;
$limit = 4;
$result = db_query(db_rewrite_sql("SELECT node.title, node.nid, node_revisions.teaser, node.changed FROM node INNER JOIN term_node ON node.nid = term_node.nid inner join node_revisions on node_revisions.nid = node.nid WHERE term_node.tid = %d AND node.status = 1 AND node.moderate = 0 AND node.promote = 1 ORDER BY node.created DESC LIMIT %d"), $tid, $limit);
while($anode = db_fetch_object($result)) {
$changed = date('j. M Y', $anode->changed);
$items[] = "<h3>".l($anode->title, "node/". $anode->nid) . " ($changed)"."</h3>". "<p> $anode->teaser </p>";
}
if(!$items) {
echo "no news actually";
} else {
echo theme_item_list($items);
}
?>

postol som to sem priamo lebo tu par ludi povedalo ako ale neukazali priklad (kedze 1x vidiet > 100x pocut resp citat o) a aj mne tym predlzili hladanie o neaku dobu, dufam ze inym pomoze ;) (celkom by sa tu zisiel WYSIWYG editor postovanie niektorych prispevkov by trvalo kratsie)

viete niekto o neakej citlivosti resp slabych miestach (ako inak prelozite do ludkeho jazyka slovenskeho vulnerability) co by to mohlo mat? (ten snippet s ktoreho som vychadzal nemal komentar od moderatora o tom ze by bol nebezpecny ale jemu velmi podobny zo skorsieho obdobia ano)
