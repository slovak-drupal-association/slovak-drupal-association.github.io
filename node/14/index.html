Inštalujeme Drupal

14 november, 2005 - 15:43 poslal mirek

Prvý ozajstný článok na tomto portále by som rád venoval procesu, ktorým musí prejsť každý, kto chce CMS Drupal naozaj nasadiť a používať. Áno, hovorím o inštalácii. Takže? Poďme na to!

Pred samotným začiatkom však musím upozorniť, že sa budem sústreďovať na OS Linux. Postup na rozchodenie pod Windowsom však bude dosť obdobný (aj tak reálne nasadenie bude v skutočnosti najčastejšie na webovom serveri Apache pod Linuxom). Čo teda budeme potrebovať?

  • fungujúci webový server Apache
  • shell konto na serveri, kde beží webový server (stačí však aj FTP prístup)
  • fungujúci SQL server (MySQL, PostgreSQL)
  • admin práva na SQL serveri na vytvorenie novej databázi alebo mať k dispozícii vytvorenú databázu a právo na vytváranie nových tabuliek
  • pripojenie do Internetu
  • niečo dobré na pitie (čo to tu máme... hmm... môj mokrý sen... :-))) a môže byť aj niečo dobré na papanie (u mňa však ostanem iba pri pití :-/)

Pokiaľ máme všetko horeuvedené, môžeme začať s inštaláciou. Začneme s tým, že si stiahneme a rozbalíme základný balík s inštaláciou.

Zo stránky www.drupal.org si teda stiahneme základný inštalačný balík s Drupal-om (v súčasnosti drupal-4.6.3.tar.gz). Na webovom serveri sa presunieme do miesta, kde ho chceme rozbaliť (v podstate náš Drupal-ovský www root) a rozbalíme ho príkazom

tar xvzf drupal-4.6.3.tar.gz

Je však lepšie ho rozbaliť o adresár vyššie, ako bude náš www root, pretože po rozbalení budú všetky súbory v adresári drupal-4.6.3. Tento potom bude treba iba premenovať na názov nášho www root-a. Toto však nemusí byť vždy možné (nemusíme mať na to dostatočné práva). V takom prípade balík rozbalíme a obsah spomínaného adresára presunieme do nášho www root-a (mv drupal-4.6.3/* drupal-4.6.3/.htaccess .).

V adresári sa bude nachádzať aj súbor s názvom .htaccess. S jeho nastavením sa zaoberať nebudeme. Preto, aby nerobil problémy, ho môžete premenovať príkazom

mv .htaccess .htaccess.old

Pokiaľ sa pokúsite v tomto momente pozrieť výsledok vašej inštalačnej práce, budete zrejme sklamaní. Žiadny sprievodca inštaláciou ako napr. v CMS CivicSpace alebo Mambo tu nie je (myslím, že som niekde čítal, že by mal byť dostupný od verzie 4.7, ale nie som si istý...). Uvidíte iba chybovú hlášku o zakázanom prístupe k MySQL serveru.

Ďalej môžeme pokračovať podľa návodu, ktorý je uvedený v súbore INSTALL.txt. Takže podľa návodu je naším ďaľším krokom vytvorenie databázy pre Drupal.

Aby všetko fungovalo správne, je nutné vytvoriť novú SQL databázu (budeme uvažovať, že máme prístup na MySQL server), kde budú uložené všetky dáta z Drupal-u. To zabezpečíme príkazom

mysqladmin -u user -p create drupal

kde user je používateľské meno používateľa, ktorý má právo na GRANT a CREATE operácie. Po zadaní tohto príkazu budete vyzvaní, aby ste zadali heslo (používateľa user) a bude vytvorená nová databáza s názvom drupal.

To však nie je všetko. V ďaľšom kroku je potrebné nastaviť prístupové práva pre túto databázu. Preto sa treba prihlásiť:

mysql -u user -p

Po zadaní hesla zadáte z príkazového riadku MySQL nasledujúci príkaz:

GRANT ALL PRIVILEGES ON drupal.* TO nobody@localhost IDENTIFIED BY 'password';

kde:

  • drupal je názov databázy
  • nobody@localhost je používateľské meno vášho MySQL konta webového servera
  • password je heslo potrebné pre prihlásenie

Pre aktiváciu práv je ešte nutné zadať príkaz:

flush privileges;

Tým by sme mali mať vytvorenú databázu a nastavené prístupové práva. Ďalej potrebujeme túto databázu naplniť tabuľkami potrebnými pre fungovanie Drupal-a. Presunieme sa preto do adresára, kde sme rozbalili súbory s Druapl-om a z príkazového riadku napíšeme:

mysql -u nobody -p drupal < database/database.mysql

Po tomto kroku máme k dispozícii nainštalovaný Drupal spolu s fungujúcou databázou. K tomu, aby nám tieto dve veci fungovali spolu, je potrebné ešte správne nakonfigurovať konfiguračný súbor Drupal-u.

Vo svojom najobľúbenejšom textovom editore (Vim) otvoríme súbor sites/default/settings.php a nájdeme nasledovný riadok:

$db_url = 'mysql://username:password@localhost/database';

Jeho význam je nasledovný:

  • mysql znamená, že používame MySQL server
  • username je prihlasovacie meno na MySQL server (nobody)
  • password je heslo pre používateľa s prihlasovacím menom username (nobody)
  • localhost je adresa MySQL servera, kde sa nachádza naša databáza (localhost znamená, že webový aj sql server je jeden a ten istý stroj)
  • database je názov databázy, kde sa nachádzajú všetky tabuľky pre Drupal-a

Pre úplnosť je ešte nutné upraviť riadok

$base_url = "http://www.example.com";

kde treba uviesť úplnú cestu k vašej www stránke s Drupal-om.

Po uložení tohto súboru máte váš CMS systém Drupal nainštalovaný. Takže do vášho najobľúbenejšieho www prehliadača (Firefox) zadajte adresu vašej stránky a môžete začať s administráciou a počiatočnou konfiguráciou. O tom ale zasa niekedy inokedy... ;)

Snáď len jedna linka - o niečo komplexnejší návod na to, ako úspešne nainštalovať Drupal, nájdete na stránkach LinuxExpress-u. Tiež som ho ešte nečítal... ;)
